主要设计思路：
一	分层：
1	其中下层只能被直接上层所调用
2	层与层之间的数据交互(如果有的话)：数据对象放入直接下层(该对象要加锁)。
3	有些层存在一些本层的数据，则该对象作为该层线程的局部变量，只供该线程读写，不用加锁。

二	每层的模块							
GcClient主要处理流程分为七层
(0)MCoreMgr 	模块	主要流程入口及管理模块
(1)MListen	模块	负责监听
(2)MRcvOpen	模块	负责收用户的协议信息,根据协议填充数据
(3)MCONN	模块	负责异步连接GcS(或者代理服务器)
(4)MAGENT	模块	负责代理的认证(如果用户通过代理连接)
(5)MHAND	模块	负责与GcS握手建立
(6)MTrans	模块	负责数据转发
其中下层模块仅供直接上层模块调用
