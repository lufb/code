主要设计思路：
一	分层：
1	其中下层只能被直接上层所调用
2	层与层之间的数据交互(如果有的话)：数据对象放入直接下层(该对象要加锁)。
3	有些层存在一些本层的数据，则该对象作为该层线程的局部变量，只供该线程读写，不用加锁。

二	每层的模块							
(0)MCoreMgr 	模块	主要流程入口					供GcClient调用				
(1)MListen	模块	负责监听					监听线程，把监听到的数据加到下层中					
(2)MConnect	模块	负责异步connect以及发送Hello包			当连接完成后，将套接字对放到下层中
(3)MHand	模块	负责握手建立(收hello包，收open包，回复open包)	处理握手，握手完成后，传入下层
(4)MTrans	模块	负责数据转发					数据转发
